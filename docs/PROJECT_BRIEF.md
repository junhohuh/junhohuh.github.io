# Project brief

Derived from [PROJECT_INTAKE.md](PROJECT_INTAKE.md). Update this file when scope or tone changes so tools stay aligned.

## Audience

- **Owner:** you (edit content in-repo).
- **Readers:** recruiters and hiring managers opening your link from applications.

## Product (MVP)

1. Clear **landing** page introducing you.
2. **About** with short bio, contact, and **résumé highlights** plus link to full PDF.
3. **Projects** section with name, summary, tech, links.
4. **Résumé** available as a prominent PDF link (`/resume.pdf`).
5. **Polish:** readable type, spacing, responsive layout; professional with a little personality.

## Non-goals (for now)

- Native mobile apps.
- Blog/CMS, comments, accounts, database-backed CMS for MVP.

## Tone

Professional and structured; not stiff—leave room for a bit of personality.

## Success

Recruiters can understand who you are, what you built, and how to contact you quickly; site works on desktop and mobile browsers.

## Stack (current)

Astro + TypeScript + Tailwind, **static** deploy (`dist/`). See [ARCHITECTURE.md](ARCHITECTURE.md).
